Memory-MCP

Self-hosted MCP memory platform for coding agents and assistants.

Memory-MCP combines structured memory, hybrid search, document knowledge base, knowledge graph, extraction pipeline, cross-session context, and reliability controls into one MCP-compatible backend.

Project overview Installation Environment presets Docker operations

Core capabilities

  • Hybrid retrieval: keyword + semantic search.
  • Knowledge graph: triples, neighbors, path discovery.
  • Knowledge base: ingest/search text, file, and URL content.
  • Cross-session context and conversation history.
  • Memory lifecycle: consolidation, decay, merge, TTL cleanup.

MCP tool surface

  • memory_search, memory_upsert, memory_list
  • kb_add_document, kb_search_documents
  • kg_add_triple, kg_get_neighbors, kg_find_path
  • extract_memories, search_extracted_memories
  • cross_session_start, cross_session_context

Reliability and safety

  • Per-tool rate limiting and request metrics.
  • LLM circuit breaker and deterministic fallback.
  • Health snapshots via memory_health.
  • CI quality gates (Ruff, mypy, pytest).

Quick start

Local: cp .env.local .env then python -m mcp_server.server

Docker: ./docker-compose.sh start, cp .env.docker .env, then run server.

See install guide for full deployment options.
